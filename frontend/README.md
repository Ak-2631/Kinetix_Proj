# Kinetix: AI-Driven Post-Op Rehabilitation Portal

Kinetix is a fully functional Minimum Viable Product (MVP) built for rapid, modern post-operative rehabilitation tracking. It leverages AI-driven computer vision, gamification, voice navigation, and intelligent summaries to create an intuitive recovery experience for both patients and doctors.

## 🚀 Key Features

*   **RehabVision™:** Uses Google's **MediaPipe PoseLandmarker** and `react-webcam` to track human anatomy. Calculates knee flexion angles in real-time, automatically counting successful reps when the patient hits the target depth (<90°).
*   **Gamification:** Features an animated "Recovery Velocity" ring built with `framer-motion` and dynamic milestone celebrations via `react-confetti` to keep patients motivated.
*   **Interactive Pain Map:** A bespoke interactive SVG human body map mapped securely to a sleek Shadcn-style 1-10 pain slider modal.
*   **Voice Navigation:** Hands-free workout control through the Web Speech API (`react-speech-recognition`). Say *"Start my exercises"* to toggle the live AI tracking module.
*   **Clinical Synthesis (Doctor Dashboard):** Contains a deterministic mock LLM API Route that synthesizes patient rep counts and pain history into actionable clinical evaluations for supervising physicians.

## 💻 Tech Stack

*   **Frontend:** Next.js (App Router), React 19, TypeScript
*   **Styling:** Tailwind CSS 4 (Custom dark medical UI theme), `lucide-react` icons
*   **Computer Vision:** `@mediapipe/tasks-vision`
*   **Animation:** `framer-motion`
*   **Voice:** `react-speech-recognition`
*   **Backend / API:** Native Next.js API Routes (`/api/summary`)

## 🛠️ Getting Started

First, ensure you are in the project root directory and navigate into the `frontend` folder:

```bash
cd frontend
```

Next, verify that all dependencies are installed (including `regenerator-runtime` for voice nav polyfills):

```bash
npm install
```

Start the development server:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the patient dashboard. You can access the clinical synthesis generated by the API on the [Doctor Dashboard](http://localhost:3000/doctor).
